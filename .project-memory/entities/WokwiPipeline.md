# WokwiPipeline

## Means
For Arduino briefs (29-36), real sketches + circuit diagrams live in `content/wokwi-projects/exp-N-slug/`. An author-time script (`scripts/wokwi-capture.sh`, also `npm run wokwi:capture`) compiles each sketch via `arduino-cli` and runs it headlessly in the Wokwi cloud simulator via `wokwi-cli`, capturing `screenshot.png` + `serial.log` + `run.log` to `public/wokwi-captures/exp-N/`. The committed captures are what the page renders — the static site never invokes Wokwi at build time, preserving offline-first.

## Per-project shape
```
content/wokwi-projects/exp-N-slug/
├── exp-N-slug.ino         (Arduino sketch — filename must match dir basename)
├── diagram.json           (Wokwi parts + connections)
├── wokwi.toml             (firmware/elf paths into build/)
├── meta.json              (screenshotPart, screenshotTimeMs, timeoutMs, title, summary)
└── build/                 (gitignored — regenerated by arduino-cli)
```

## Author workflow
1. Author or edit a sketch + diagram + meta.json
2. `npm run wokwi:capture` (or `--only=N` for one)
3. Review captured PNG + serial.log
4. Commit captures to repo

## Requirements
- `arduino-cli` on PATH (Linux x86_64 install from arduino.cc releases)
- `wokwi-cli` on PATH (download from github.com/wokwi/wokwi-cli releases)
- `arduino:avr` core installed: `arduino-cli core install arduino:avr`
- `WOKWI_CLI_TOKEN` env var (free tier from https://wokwi.com/dashboard/ci)
- For shells where env vars aren't exported: `eval "$(grep 'WOKWI_CLI_TOKEN=' ~/.bashrc | head -1)"`
